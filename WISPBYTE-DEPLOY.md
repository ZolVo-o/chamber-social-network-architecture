# Deploy на WispByte

## Главное правило

Не смешивай файлы из разных версий проекта.

Если WispByte собирает проект командой `npm run build`, он использует исходники из `src/` и `server/`, а не готовый `dist/index.html`.

Если загрузить новый `src/App.tsx`, но оставить старый `src/lib/api.ts`, сборка упадет с ошибкой вида:

- `getSocketConnection is not exported by src/lib/api.ts`

## Вариант 1 — рекомендованный: WispByte собирает проект из исходников

Загружай весь проект **кроме**:

- `node_modules/`
- `dist/`

Минимум проверь, что на сервер точно попали актуальные версии этих файлов:

- `src/App.tsx`
- `src/lib/api.ts`
- `src/config.ts`
- `server.js`
- `server/app.js`
- `server/routes/messages.js`
- `server/routes/rooms.js`
- `package.json`
- `package-lock.json`

Если сомневаешься, лучше перезалить весь проект целиком, чем обновлять отдельные файлы вручную.

## Вариант 2 — prebuilt deploy

Если ты хочешь загружать уже собранный фронтенд, тогда локально выполни сборку и загрузи готовый `dist/index.html` вместе с backend-файлами той же версии.

Но не смешивай этот режим с вариантом, где WispByte сам выполняет `vite build` из `src/`.

## Docker image

Рекомендуется выбрать:

- `nodejs_22`

Если его нет или хочешь новее, можно `nodejs_24`.

## Startup command

Вставь в редактор команд WispByte вот эту команду:

```bash
if [[ -d .git ]] && [[ ${AUTO_UPDATE} == "1" ]]; then git pull; fi; /usr/local/bin/npm install --include=dev && /usr/local/bin/npm run build && /usr/local/bin/npm run start
```

## Переменные окружения

Тебе нужен файл `.env.local` в корне проекта.

Сейчас для твоего сервера используется:

```env
VITE_API_BASE_URL=http://87.106.190.187:10906
```

Если адрес потом изменится, просто поменяй это значение.

## Как запускать

1. Выбери `nodejs_22`
2. Сохрани startup command
3. Загрузи проект целиком из одной версии
4. Убедись, что `.env.local` лежит в корне
5. Перезапусти сервер

## Если уже был частичный деплой

Если ты раньше загружал файлы по одному и сборка начала падать, действуй так:

1. Удали старые исходники проекта на WispByte или полностью перезапиши их.
2. Заново загрузи весь актуальный проект одной пачкой.
3. Убедись, что `src/App.tsx` и `src/lib/api.ts` обновились вместе.
4. Запусти `npm install --include=dev`.
5. Запусти `npm run build`.
6. Затем запусти приложение.

## Что должно быть в логах

Нормальный запуск выглядит примерно так:

- `npm install --include=dev`
- `vite build`
- `Centum server listening on port ...`

Если вместо этого есть ошибка импорта из `src/lib/api.ts`, значит на сервере лежат старые или смешанные исходники.

## Проверка после запуска

Открой:

- основной сайт: `http://87.106.190.187:10906`
- health-check: `http://87.106.190.187:10906/health`

Если `/health` отвечает `{"status":"ok"}`, значит сервер поднялся.
